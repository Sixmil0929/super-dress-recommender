# 项目操作流程

本文档分为两部分：

- 第一次在新电脑上初始化项目
- 平常日常启动项目

## 一、第一次初始化流程

### 第 1 步：下载并打开 Docker Desktop

先启动 Docker Desktop，并确认左下角状态为 `Engine running`。


### 第 2 步：重置并启动数据库

```bash
docker compose -f docker_compose.yml down -v
docker compose -f docker_compose.yml up -d
```

说明：

- `down -v` 会删除当前数据库容器和数据卷
- 这一步适合第一次初始化或数据库状态错误时使用

### 第 3 步：导入数据库初始化文件

```bash
docker exec -i dress_select_db psql -U postgres -d postgres < /Users/a1/Downloads/init.sql
```

### 第 4 步：验证数据库是否导入成功

```bash
docker exec -it dress_select_db psql -U postgres -d postgres -c "SELECT COUNT(*) FROM clothing_features;"
```

如果返回一个大于 `0` 的数字，说明 `clothing_features` 表已经成功恢复。

### 第 5 步：启动后端

```bash
source .venv/bin/activate
python api_server.py
```

后端启动后，可访问：

```text
http://localhost:8000/docs
```

如果能看到以下接口，说明统一后端已正常启动：

- `/api/user/register`
- `/api/user/login`
- `/api/user/profile`
- `/api/random_looks`
- `/api/recommend_by_survey`

### 第 6 步：启动前端

新开一个终端窗口，执行：

```bash
cd super-dress-recommender/frontend
npm install
npm run dev
```

浏览器打开：

```text
http://127.0.0.1:5173
```

## 三、日常启动流程

适用于：

- 数据库已经正确导入过
- 平常开发时重新打开项目

### 第 1 步：打开 Docker Desktop

确认状态为 `Engine running`。

### 第 2 步：启动数据库

```bash
docker compose -f docker_compose.yml up -d
```

说明：

- 这一步只负责启动数据库
- 日常使用时不需要重复执行 `down -v`
- 日常使用时也不需要重复导入 `init.sql`

### 第 3 步：启动后端

```bash
source .venv/bin/activate
python api_server.py
```

### 第 4 步：启动前端

新开一个终端窗口，执行：

```bash
cd super-dress-recommender/frontend
npm run dev
```

### 第 5 步：打开项目页面

```text
http://127.0.0.1:5173
```
