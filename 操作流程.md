# 项目操作流程

本文档分为两部分：

- 第一次在新电脑上初始化项目
- 平常日常启动项目

补充约定：
- `images` 文件夹应放在项目根目录
- 后端默认使用项目根目录下的 `.venv` 虚拟环境

## 一、第一次初始化流程

### 第 1 步：下载并打开 Docker Desktop

先启动 Docker Desktop，并确认左下角状态为 `Engine running`。

### 第 1.5 步：确认项目根目录文件

项目根目录建议至少包含这些内容：

```text
project-root/
├── images/
├── init.sql
├── docker_compose.yml
├── api_server.py
└── .venv/
```

其中：

- `images/` 是组长单独发的服装图片目录
- `init.sql` 是数据库初始化文件
- `.venv/` 是项目后端虚拟环境


### 第 2 步：重置并启动数据库

```bash
docker compose -f docker_compose.yml down -v
docker compose -f docker_compose.yml up -d
```

说明：

- `down -v` 会删除当前数据库容器和数据卷
- 这一步适合第一次初始化或数据库状态错误时使用

### 第 3 步：导入数据库初始化文件

Windows PowerShell 和 Mac 都可使用：

```bash
docker cp init.sql dress_select_db:/tmp/init.sql
docker exec -it dress_select_db psql -U postgres -d postgres -f /tmp/init.sql
```

说明：

- 这里默认您已经在项目根目录下执行命令
- `init.sql` 指的就是项目根目录下的数据库初始化文件

### 第 4 步：验证数据库是否导入成功

```bash
docker exec -it dress_select_db psql -U postgres -d postgres -c "SELECT COUNT(*) FROM clothing_features;"
```

如果返回一个大于 `0` 的数字，说明 `clothing_features` 表已经成功恢复。

### 第 5 步：启动后端

如果第一次没有虚拟环境，需要先创建：

```powershell
python -m venv .venv
```

如果已经有 `.venv`，则直接激活并启动后端。

Windows PowerShell：

```powershell
.\.venv\Scripts\Activate.ps1
python api_server.py
```

Mac：

```bash
source .venv/bin/activate
python3 api_server.py
```

说明：

- 每次新开一个终端，通常都要重新激活一次 `.venv`
- 如果终端前面已经有 `(.venv)`，说明虚拟环境已经激活，可以直接运行 `python api_server.py`

后端启动后，可访问：

```text
http://localhost:8000/docs
```

如果能看到以下接口，说明统一后端已正常启动：

- `/api/user/register`
- `/api/user/login`
- `/api/user/profile`
- `/api/random_looks`
- `/api/recommend_by_survey`

### 第 6 步：启动前端

新开一个终端窗口，执行：

```bash
cd frontend
npm install
npm run dev
```

浏览器打开：

```text
http://127.0.0.1:5173
```

## 三、日常启动流程

适用于：

- 数据库已经正确导入过
- 平常开发时重新打开项目

### 第 1 步：打开 Docker Desktop

确认状态为 `Engine running`。

### 第 2 步：启动数据库

```bash
docker compose -f docker_compose.yml up -d
```

说明：
- 这一步只负责启动数据库
- 日常使用时不需要重复执行 `down -v`
- 日常使用时也不需要重复导入 `init.sql`

### 第 3 步：启动后端

Windows PowerShell：

```powershell
.\.venv\Scripts\Activate.ps1
python api_server.py
```

Mac：

```bash
source .venv/bin/activate
python3 api_server.py
```

说明：

- 如果这是新开的终端窗口，需要先激活 `.venv`
- 如果当前终端前面已经显示 `(.venv)`，可以直接运行 `python api_server.py`

### 第 4 步：启动前端

新开一个终端窗口，执行：

```bash
cd frontend
npm run dev
```

### 第 5 步：打开项目页面

```text
http://127.0.0.1:5173
```
