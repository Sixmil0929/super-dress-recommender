# 脚本索引

这份文档只解释 `scripts/` 目录下的研发脚本。  
这些脚本**都不是当前网站日常运行的主入口**，当前主链路仍然是：

- 后端：[api_server.py](../api_server.py)
- 数据库：[docker_compose.yml](../docker_compose.yml) + [init.sql](../init.sql)
- 前端：[frontend](../frontend)

## 使用说明

为了便于判断脚本价值，这里给每个脚本加了一个建议状态：

- `仍可用`：逻辑上仍有用途，但不属于日常启动主链路
- `谨慎使用`：可能依赖旧数据结构、旧路径或旧环境，运行前需要先核对
- `历史脚本`：主要保留作参考，不建议直接执行

---

## 一、`scripts/db_tools/`

这类脚本与建库、导数、修数相关。

### [setup_db.py](../scripts/db_tools/setup_db.py)
- 建一个最基础的 `clothing_features` 表，并启用 `pgvector`
- 更像早期数据库打底脚本
- 当前状态：`历史脚本`
- 原因：当前正式数据库初始化已由 [init.sql](../init.sql) 接管，表结构也比这里完整

### [init_database.py](../scripts/db_tools/init_database.py)
- 创建另一套名为 `dress_recommender` 的数据库和 `clothing_inventory` 等表
- 当前状态：`历史脚本`
- 原因：和当前实际使用的 `postgres` + `clothing_features` 体系不一致

### [process_clothing_images.py](../scripts/db_tools/process_clothing_images.py)
- 从 `P1_Dataset` 读取图片和 CSV，抽取 CLIP / DINOv2 特征并写库
- 当前状态：`谨慎使用`
- 原因：仍有研究价值，但默认依赖旧数据集目录、旧数据库名和特征流程，运行前必须先改配置

### [build_private_db.py](../scripts/db_tools/build_private_db.py)
- 面向私有服装库的一体化建库脚本
- 当前状态：`历史脚本`
- 原因：文件内大量内容已被注释，更像阶段性实验稿

### [build_private_db_p2345.py](../scripts/db_tools/build_private_db_p2345.py)
- 面向 `P2-P5` 数据集的建库脚本
- 当前状态：`历史脚本`
- 原因：和当前比赛演示链路无关，且同样带有重实验性质

### [manual_fix.py](../scripts/db_tools/manual_fix.py)
- 手工修正特定图片标签
- 当前状态：`仍可用`
- 适用场景：发现个别图片的 `style / scene / season / item_category` 等标签明显错误时，可人工覆写

## 二、`scripts/debug/`

这类脚本主要用于查库、验数、辅助排错。

### [query_database.py](../scripts/debug/query_database.py)
- 数据库查询与向量相似度检索工具
- 当前状态：`谨慎使用`
- 原因：保留研究价值，但默认连接的是另一套旧数据库配置，直接运行前要核对

### [check_db.py](../scripts/debug/check_db.py)
- 通过关键词搜索数据库里的衣服，并可视化查看标签
- 当前状态：`仍可用`
- 适用场景：抽查某批图片的标签和数据库记录是否一致

### [check_metadata.py](../scripts/debug/check_metadata.py)
- 打印数据库里部分衣服的品牌、价格、性别、标签等元数据
- 当前状态：`仍可用`

### [tag_analytics.py](../scripts/debug/tag_analytics.py)
- 对全库 `clip_tags` 做整体统计
- 当前状态：`仍可用`

### [view_db.py](../scripts/debug/view_db.py)
- 打印数据库前几条衣服记录
- 当前状态：`仍可用`

### [tag_inspector.html](../scripts/debug/tag_inspector.html)
- 一个独立 HTML 调试页面
- 当前状态：`历史脚本`
- 原因：不属于当前前端产品的一部分，更像离线辅助查看页面

## 当前建议

如果只是为了维护和继续开发网站，请优先关注这些：

1. [api_server.py](../api_server.py)
2. [frontend](../frontend)
3. [docker_compose.yml](../docker_compose.yml)
4. [init.sql](../init.sql)
5. [scripts/db_tools/manual_fix.py](../scripts/db_tools/manual_fix.py)（如需修标签）
6. [scripts/debug](../scripts/debug) 下的查库脚本（如需排错）

其余大多数脚本当前都更适合作为：

- 研究记录
- 历史实现参考
- 后续复现素材

而不是比赛阶段的主维护入口。
